MVCApplication

new MVCApplication(prefs) → {MVCApplication}

Объект MVC-Приложение.

Поддерживает коллекции MVC-объектов, предоставляет интерфейс для управления ими (создание, удаление, поиск и получение их из коллекций) и отвечает за формирование главного окна - основного родительского диалога. Кроме того, может в целом выступать в качестве изолированного пространства имён для всего пользовательского приложения.

addController(param) → {MVCController}

Добавление нового Контролёра.

Метод создаёт и добавляет в коллекцию controllers объект MVCController (контролёр). В качестве аргумента вызова принимает комплексный параметр param, который передаётся конструктору MVCController(param), который выполняет связывание Модели и Представления.

ПРИМЕЧАНИЕ: Задавать значения параметров id и app необязательно, метод обеспечивает автоматическую инициализацию свойства свойства param.app ссылкой на собственный объект MVCApplication и, в случае отсутствия значения param.id - обеспечивает автоматическую генерацию уникального id для Контролёра.

Parameters:
Name Type Description
param cparams

литерал объекта, представляющий параметры инициализации контролёра;

Properties
Name Type Argument Default Description
app string <optional>

автоматически инициализируется ссылкой на MVCApplication, в контексте которого происходит вызов этого метода;
id string <optional>

id для создаваемого контролёра, должен быть уникальным в рамках коллекции controllers, если не указан - генерируется автоматически;
binding string <optional>

строка биндинга (см. MVCController);
bind boolean <optional>
 true

флаг, связывания (см. MVCController).
Returns:
Type
MVCController
Example

Пример создания Контролёра

// Значение myModel.value.text будет сразу присвоено myView.text:
myApp.addController({ binding:"myModel.value.text:myView.text" });
// ...
// Значение myModel.value.text не будет сразу присвоено myView.text:
myApp.addController({ binding:"myModel.value.text:myView.text", bind:false });

addDocument(obj) → {MVCDocument|null}

Создаёт и добавляет новый Документ в коллекцию документов приложения MVCApplication.documents

Для получения нового Документа используется вызов метода MVCApplication.CreateDocument(),

При добавлении нового Документа с ним ассоциируется специально создаваемая Модель, имеющая единственное свойство document, указывающее на созданный экземпляр MVCDocument и значение id такое же как id созданного Документа.

Кроме того, в коллекцию представлений приложения (MVCApplication.views) добавляется ссылка на представление документа, при этом свойство id представления документа устанавливается в соответствии со значением свойства id Модели Документа: model.id == view.id == MVCDocument.mainView.id.

ПРИМЕЧАНИЕ:

 * Расширение и определения функциональности самого объекта документа следует выполнять в рамках метода
   MVCApplication.CreateDocument(), путём его переопределения;
 * В случае успешного создания документа на него переустанавливается указатель MVCApplication.activeDocument, 
   в противном случае - его текущее значение не изменяется.
Parameters:
Name Type Argument Description
obj object <optional>

литерал объекта, которым будет расширен базовый объект MVCDocument, полученный с помощью вызова фабричного метода MVCApplication#CreateDocument()
Returns:

В случае успеха возвращается созданный объект MVCDocument, в противном случае - null.

Type
MVCDocument | null

addModel(obj) → {MVCCModel}

Добавление Модели:

Создаёт и добавляет объект-модель в коллекцию models приложения. В качестве аргумента метод принимает литерал классического JavaScript-объекта, который выступает в качестве основы объекта данных приложения - модели. В литерале предусматривается несколько специально-зарезервированных полей, определяющих особенные характеристики модели.

Parameters:
Name Type Argument Description
obj object <optional>

литерал, представляющий пользовательские данные - основу объекта MVCModel

Properties
Name Type Argument Description
id string <optional>

id (идентификатор) объекта, должен быть уникальным в пределах коллекции;
validator function <optional>

метод, выполняющий валидацию модели (см MVCModel.validator());
Returns:
Type
MVCCModel
Example

Стандартный способ создания Моделей:

// myApp – созданный ранее объект MVCApplication
var myData = myApp.addModel({
    id:"myData", 
    value:{ txt:"Мои данные" },
    print:function() { $.writeln(this.value.txt) },
    // метод-валидатор - предусмотрен библиотекой, 
    // может определяется в конкретном экземпляре 
    // Модели при необходимости (см. MVCModel.validator();)
    validator:function(key, oldVal, newVal, ctrl) {
        return (newVal.length < 10 || newVal.length > 30) ? false : true;
    }
});
myData.print(); // => `Мои данные`
$.writeln(myData instanceof MVCModel); // => true
$.writeln(typeof myData); // => object
$.writeln(myData.id == "myData"); // => true
$.writeln(myData.value.txt == "Мои данные"); // true

Метод создаёт и добавляет объект-представление в коллекцию views Приложения. В литерале предусматривается несколько специально-зарезервированных полей, определяющих особенные характеристики создаваемого объекта-представления.

Example

Полная сигнатура метода .addView()

// myApp – созданный ранее объект MVCApplication
var myView = myApp.addView({ 
   id:"myView",     // id объекта 
   // родительский контейнер для ел.уп.
   parent:myApp.window.panel,

closeDocument(doc) → {MVCDocument}

Закрытие документа.

Процесс закрытия реализует удаление главного Представления и Модели, представляющих Документ, из соответствующих коллекций Приложения и переустановку указателя MVCApplication.activeDocument на другой ближайший в коллекции объект документа.

Вся специальная работа, связанная с удалением данных, представлений и пр... созданных в рамках документа, может быть реализована в методе .close() реального экземпляра Документа. Данный метод должен возвращать true в результате успешного выполнения, в противном случае операция удаления документа в рамках данного метода (closeDocument()) прерывается (Представление, Модель и Контролёр, связанные с Документом - удалены не будут).

Удаление документа начинается с вызова его собственного метода .close(), после чего происходит удаление связанных с ним основных MVC-объектов из соответствующих коллекций Приложения.

Parameters:
Name Type Argument Default Description
doc string | MVCDocument <optional>
 activeDocument

Ссылка на удаляемый документ или его id. Если не указано - удаляется текущий активный документ (определяется по значению свойства activeDocument)
Returns:

Возвращает текущее значение activeDocument с учётом выполнения закрытия документа.

Type
MVCDocument

CreateDocument() → {MVCDocument}

Фабричный метод для создания экземпляров Документов.

По умолчанию метод возвращает базовый объект MVCDocument, привязанный к данному Приложению (выполняется вызов new MVCDocument(this), где this указывает на контекст Приложения). Чтобы определить необходимую функциональность базового объекта MVCDocument, следует переопределить данный метод в пользовательском объекте Приложения, при этом, метод должен возвращать созданный экземпляр Документа или null, в случае проблем его создания.

ПРИМЕЧАНИЕ: Данный метод автоматически используется для получения объекта Документа при вызове методов MVCApplication.addDocument() и MVCApplication.loadDocument().

Type
MVCDocument

CreateMainView(rcString) → {MVCView}

Приватный метод, формирующий главное окно приложения.

Метод формирует главное окно приложения из ресурсной ScriptUI-строки, переданной в качестве аргумента вызова. При формировании окна обеспечивается корректная обработка свойства properties:true, если оно заявлено в ресурсной строке и навешивается обработчик на событие 'close' окна, который обеспечивает вызов обработчика Приложения .onExit(), который может быть определён для выполнения каких либо операций при закрытии главного окна приложения. В результате работы, метод инициализирует свойства .window и .mainView объекта-приложения.

Конструктор MVCApplication() вызывает этот метод при создании экземпляра Приложения. Подходящее место для явного вызова этого метода - тело функции Init(), предусмотренной для переопределения в "рабочем" экземпляре объекта-приложения.

Parameters:
Name Type Description
rcString string

ресурсная ScriptUI-строка для формирования диалога, представляющего главное окно приложенияю
Returns:

Главное окно Приложения.

Type
MVCView
Example

CreateMainView() позволяет получить полный контроль над процессом создания главного окна приложения

var myApp = new MVC.Application();
myApp.Init = function() {
   // ресурсная строка любой длинны и сложности
   var rc = "dialog { preferredSize:[350,200], properties:{resizeable:true} }"
   this.CreateMainView(rc);
   // В результате вызова CreateMainView() свойству this.window присвоена ссылка
   // на созданное окно, теперь с ним можно работать напрямую
   this.window.text = "la-la-la...";
   //... 

   // делать здесь вызов this.window.show() не нужно, - этот вызов будет
   // произведён внутри метода .run()
};

myApp.run(); // Выполнит .Init() и откроет окно (выполнит .window.show();)

findController(obj) → {MVCController|undefined}

Поиск связанного с объектом Контролёра.

Происходит попытка обнаружения контролёра, связанного с аргументом. В качестве аргумента могут выступать либо MVC-объекты, либо ассоциированные с ними объекты (ScriptUI-объекты графического интерфейса UIControl-s, или ассоциированные с Моделями объекты). В случае успешного нахождения контролёра - возвращает ссылку на него из коллекции контролёров приложения, в противном случае - возвращает undefined.

Метод удобно использовать для получения ссылки на контролёр из тел обработчиков элементов управления или объектов-моделей, указывая методу в качестве аргумента вызова своё свойство this.

Parameters:
Name Type Description
obj object

MVCModel | MVCView | any - или любой объект, ассоциированный с имеющимся MVC-объектом
Returns:

Возвращает ссылку на объект MVCController, в противном случае, если не найдено ни одного контролёра, ассоциированного с аргументом - возвращает undefined.

Type
MVCController | undefined
Examples

Любой из вызовов поволит обнаружить ассоциированный контролёр:

myApp.findController(myModel); 
myApp.findController(myModel.value);
myApp.findController(myView);
myApp.findController(myView.control);

Получение контролёра, в обработчике элемента управления, связанного с Представлением. В данном примере создаётся модель, для которой определяется обработчик onChange - в происходит получение ссылки на контролёр и от него - на модель, после чего определяется состояние модели и в зависимости от него меняется цвет текста

myApp.addView({ 
    id:"et", 
    view:"edittext { characters:30 }", 
    control:{
        onChanging:function() {
            // получаем ссылку на контролёр, чтобы получить ссылку
            // на модель, связанную с данным представлением:
            var ctrl = myApp.findController(this),
                gfx = this.graphics;
            // в зависисмости от статуса модели меняем цвет текста:
            if (ctrl.model.isValid()) { 
                gfx.foregroundColor = gfx.newPen (0, [0,0,0], 1); // gfx.PenType.SOLID_COLOR == 0
                this.helpTip = "Всё хорошо";        
            } else {
                gfx.foregroundColor = gfx.newPen (0, [1,0,0], 1);
                this.helpTip = "Неправильное значение");
            }
        }
    }
});

loadDocument() → {MVCDocument}

Загружет документ из файла.

Метод обеспечивает создание диалогового окна для открытия файлов, настроенное на заданный фильтр файлов, представленный в свойстве MVCDocument.filters - данное свойство, при создании документа инициализируется данными из одноимённого свойства приложения MVCApplication.filters и в дальнейшем, в рамках данного документа может быть независимо расширено.

Загрузка документа обеспечивается в три этапа:

1. открытие диалога для выбора файла; 
2. создание пустого объекта-Документа путём вызова MVCApplication.addDocument();
3. инициализация свойства .file у полученного на этапе (2) Документа значением, полученным на этапе (1)
   и вызов в контексте данного объекта-Документа его собственного метода MVCDocument.load().

В случае успешного открытия происходит переустановка указателя MVCApplication.activeDocument на созданный экземпляр документа и возвращается ссылка на данный экземпляр. В случае, если в окне выбора файла было нажато "Отмена" или метод документа .load() вернул false - Документ создан не будет, Значение указателя MVCApplication.activeDocument не изменится и методом будет возвращено значение null.

Таким образом, для реализации открытия файла предполагается, что метод Документа MVCDocument.load() должен быть определён, так как реальная работа по загрузке данных из файла должна обеспечиваться непосредственно собственным методом load() конкретного экземпляра Документа.

Returns:

Возвращает ссылку на открытый документ или null в случае ошибки открытия или выбора "Отмена" в диалоговом окне выбора файлов.

Type
MVCDocument

registerDocumentFactory(docsView) → {MVCApplication}

Регистрация "фабрики" документов.

Вызов метода обязателен для подготовки объекта Приложения к работе с многодокументоной архитектурой. В процессе регистрации объект Приложения будет расширен рядом дополнительных свойств:

  • documents:{Collection} - коллекция Документов в рамках приложения;
  • activeDocument:{MVCDocument} - указатель на активный объект MVCDocument;
  • documentsView:{MVCView} - указатель на Представление-контейнер для представлений Документов. Кроме того, прототип MVCApplication расширяется рядом методов (см. MVC.DOM) для работы с объектами MVCDocument

    В качестве необязательного параметра метод получает ссылку на объект MVCView - который в последствии используется как родительский контейнер для представлений всех документов, создаваемых в рамках Приложения. Для указания контейнера, его необходимо предварительно создать с использованием в качестве значения свойства id строки "Documents" и добавить в общую коллекцию представлений MVCApplication.views.

После определения представления, ссылка на него присваивается свойству приложения MVCApplication.documentsView.

removeMVC(obj) → {boolean}

Универсальный метод удаления MVC-объектов.

Принимает либо строковое значение, либо MVC-объект (Контролёр, Модель или Представление). Любое строковое значение рассматривается как возможное значение id-MVC-объекта и происходит попытка найти данный объект в коллекциях:

в начале происходит поиск в коллекции controllers, затем в models и затем в views. Далее происходит процедура удаления по следующему принципу: 

- При удалении Контролёра - удаляются связанные с ним объекты Модель и Представление.
- При удалении Модели - удаляются связанные с ней объекты-Представления и Контролёры.
- При удалении Представления - удаляется только указанный объект-Представление.

Метод удобно применять для совместного удаления связки Модель + Представление + Контролёр, передавая в качестве аргумента указатель на объект-контролёр.
Parameters:
Name Type Description
obj string | MVCController | MVCModel | MVCView

удаляемый MVC-объект (или id объекта, тогда происходит автоматическая попытка его поиска и удаления из соответствующей коллекции.)
Fires:
  • MVCView#event:onRemove
Returns:

true если удаление прошло успешно, иначе false.

Type
boolean
Example

Удаление MVC-объектов, созданных в предыдущих примерах:

// удаление только представления:
myApp.removeMVC(myApp.getViewByID("myView"));
// удаление модели также приводит и к удалению
// контролёра, все вызовы равнозначны: 
myApp.removeMVC("myModel"); // будет произведён
                           // поиск и удаление
                           // объекта со значением
                           // id == 'myModel'
// прямое удаление модели:
myApp.removeMVC(myApp.getModelByID("myModel"));
// удаление с указанием контролёра - удалит
// все три MVC-объекта:
myApp.removeMVC(myApp.findController(myModel));
// или:
myApp.removeMVC(myApp.findController(myView));
