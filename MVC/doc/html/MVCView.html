Class: MVCView

MVCView

new MVCView(id, control, rcString) → {MVCView}

MVC-Объект Представление (Вид).

Представление (или в другой терминологии – Вид) служит для отображения и управления (редактирования) данных, представленых в Моделях. По сути, объекты MVCView является обёрткой вокруг ScriptUI-объектов, представляющими элементамы управления пользовательского графического интерфейса - объекты UIControl.

'Родной' конструктор Представлений не предусматривает создание графического эл. управления, он выполняет лишь 'заворачивание' уже имеющегося элемента в необходимую 'обёртку'. А вот метод Приложения .addView({...)} напротив - всегда приводит к созданию нового элемента управления (объекта UIControl) в контексте тут же созданного объекта MVCView

Таким образом, создания Представлений с использованием конструктора MVCView() требуется в случае, когда нам необходимо существующий элемент управления «обернуть» в объект-Представление. После этого, как и в случае с Моделями, перед связыванием этого представления с моделью, его необходимо явно добавить в коллекцию представлений Приложения (MVCApplication.views)

Example
// Пусть главное окно приложения изначально включает 
// поля ввода (два элемента EditText):
var myApp = new MVCApplication({
    view:"dialog { et1:EditText{}, et2:EditText{} }"
});
var w = myApp.window;
var myView = new MVCView("et1", w.et1);
$.writeln(myView.id); // => `et1`
$.writeln(myView.view); // => `edittext {textDirection:null,type:edittext}`
$.writeln(myView.control === w.et1); // => true

// Значение id не обязательно должно совпадать с локальным
// именем элемента...

Methods

<abstract> Init() → {any}

Инициализация Представления.

Метод вызывается при создании Представления в рамках метода приложения MVCApplication.addView(...). Вызов происходит в контексте элемнта управления (в контексте элемнта свойства .control). Метод предназначен для переопределения с целью выполнения дополнительных действий по настройке графического элемента управления сразу после его создания.

Returns:

определяется пользователем.

Type
any
Example
// добавит к главному окну приложения элемент управления ListBox
// после чего сразу сформирует в нём два пункта, включающие
// изображения, загружаемые из файлов на диске:
myApp.addView({ 
    view:"listbox { characters: 20 }", 
    Init:function() {
        with(this.add(“item”, “Строка 1”)) { image = ScriptUI.newImage(file_1)};
        with(this.add(“item”, “Строка 2”)) { image = ScriptUI.newImage(file_2)};
    }
});

rebind(model, view_key, model_key) → {MVCView}

Переключает Представления на заданную Модель.

После переключения все события, связанные с изменением свойства view_key данного представления будут приводить к соответствующему изменению свойства model_key в указанной модели model.

Переключение происходит только со стороны представления, тоесть, события связанные с изменением свойства модели, которая ранее являлась связанной с данным представлением, по прежнему будут отражаться на данном представлении.

Parameters:
Name Type Argument Default Description
model MVCModel | object

Модель, на которую производится переключение;
view_key string <optional>
 "text"

Имя свойства представления, значение которого будет привязываться к заданному свойству модели. Если не задано, подразумевается, что это свойство text (типично для StaticText и EditText)
model_key string <optional>
 ""

Имя свойства модели, если не задано, извлекается из свойства binding объекта-контролёра, связанного с данной моделью. Если аргумент model не является объектом MVCModel - аргумент model_key должен быть обязательно определён.
Returns:

this - позволяет использовать цепочки вызовов.

Type
MVCView
Example

remove() → {boolean}

Приватный метод удаление объекта UIControl, представленного в свойстве control Представления.

Удаляет из родительского контейнера элемент управления, инкапсулированный в данном Представлении. Как правило, специальный вызов данного метода не требуется. Вызов метода происходит автоматически в процессе удаления самого объекта Представления - в результате вызова методов Приложения - .removeView() или .removeMVC().

Вызов данного метода приводит к срабатыванию обработчика onRemove() в контексте данного объекта MVCView.

Fires:
  • MVCView#event:onRemove
Returns:

в результате успешного удаления возвращает true, в противном случае false.

Type
boolean

unbind() → {MVCView}

«Отвязывает» Представление от Модели.

В результате вызова данного метода, любые события изменения данных в Представлении перестают отражаться на связанной с ним Модели. При этом, связанное свойство представления очищается (например, если в качестве представления выступал компонент EditBox - его свойству .text присваивается пустая строка "", а свойству .selection для элементов-списков, таких как ListBox, DropDownList и TreeView - присваивается null).

Returns:

this - позволяет использовать цепочки вызовов.

Type
MVCView
Example
myView.unbind();

Events

onRemove

Вызов данного обработчика происходит в момент удаления объекта MVCView непосредственно перед удалением графического элемента управления, представленного в данном объекте MVCView (как правило, в результате удаления самого Представления). По умолчанию данный обработчик никаких действий не выполняет.

Example

При удалении Представления в консоль будет выдано сообщение. Обработчик определяется в контсексте объекта MVCView, а не в контексте его элемента управления MVCView.control: {UIControl}

myApp.addView({
   onRemove:function() {
       $.writeln("Удаление:"+this.id)
   }
});
